Configuration of this builder. Available settings (case-\/sensitive)\+:
\begin{DoxyItemize}
\item "{}comment\+Style"{}\+: "{}\+None"{} or "{}\+All"{}
\item "{}indentation"{}\+: "{}$<$anything$>$"{}.
\item Setting this to an empty string also omits newline characters.
\item "{}enable\+YAMLCompatibility"{}\+: false or true
\item slightly change the whitespace around colons
\item "{}drop\+Null\+Placeholders"{}\+: false or true
\item Drop the "{}null"{} string from the writer\textquotesingle{}s output for null\+Values. Strictly speaking, this is not valid JSON. But when the output is being fed to a browser\textquotesingle{}s Java\+Script, it makes for smaller output and the browser can handle the output just fine.
\item "{}use\+Special\+Floats"{}\+: false or true
\item If true, outputs non-\/finite floating point values in the following way\+: NaN values as "{}\+Na\+N"{}, positive infinity as "{}\+Infinity"{}, and negative infinity as "{}-\/\+Infinity"{}.
\item "{}precision"{}\+: int
\item Number of precision digits for formatting of real values.
\item "{}precision\+Type"{}\+: "{}significant"{}(default) or "{}decimal"{}
\item Type of precision for formatting of real values.
\item "{}emit\+UTF8"{}\+: false or true
\item If true, outputs raw UTF8 strings instead of escaping them.
\end{DoxyItemize}
